============================================================
  KEY CONCEPTS
============================================================

  ISOLATION:
  - Each client has its OWN PostgreSQL database
  - Each client has its OWN Docker network
  - Client 1 CANNOT access Client 2's data (and vice versa)
  - Even if a hacker compromises Client 1, Client 2 is safe

  PERSISTENCE:
  - Database files are stored in client{N}/data/
  - Uploaded files are stored in client{N}/odoo-data/
  - Even if Docker containers are deleted, data survives

  CUSTOM MODULES:
  - Each client has its own addons/ folder
  - Different clients can have different modules installed
  - Modules are mounted at /mnt/extra-addons in the container

============================================================
  FOLDER STRUCTURE
============================================================

  odoo-saas-demo/
  |
  +-- docker-compose.yml      (defines all containers)
  +-- .env                    (passwords & config)
  +-- nginx/
  |   +-- default.conf        (reverse proxy rules)
  +-- prajatha/
  |   +-- addons/             (custom Odoo modules)
  |   +-- data/               (PostgreSQL database files)
  |   +-- odoo-data/          (uploaded files & filestore)
  +-- farmer/                 (same structure)
  +-- client3/                (same structure)
  +-- client4/                (same structure)
  +-- client5/                (same structure)
  +-- backups/                (database backup files)
  +-- start-all.bat           (start everything)
  +-- stop-all.bat            (stop everything)
  +-- reset-all.bat           (delete everything)
  +-- README.md               (documentation)
  +-- DEMO-SCRIPT.md          (presentation guide)
  +-- TROUBLESHOOTING.md      (problem solving)
